<ChangelogCard type="feat" title="主题切换圆形扩散动画" date="2026-04-14">
切换主题时，从底部状态栏主题按钮位置向外以圆形扩散过渡，取代瞬间变色。基于 View Transitions API，不支持的浏览器自动降级，`prefers-reduced-motion` 时也不触发。

```css
@keyframes theme-reveal {
  from { clip-path: circle(0% at 95% 98%) }
  to   { clip-path: circle(150% at 95% 98%) }
}
```

切换 latte → mocha → old-hope 时效果尤其明显。
</ChangelogCard>

<ChangelogCard type="fix" title="CSS 变量补全 + 动画系统统一" date="2026-04-14">
修了两个潜伏已久的问题：

1. `--c-teal` / `--c-sapphire` 在 Token 图表里被引用但三套主题都没有定义，颜色一直 fallback 到 `currentColor`。补全了 latte / mocha / old-hope 三套值。

2. `IinaTokensPage`、`CodeStatsWidget`、`TerminalPet` 各自内联了几乎相同的 `@keyframes`。全部抽离到 `css/tailwind.css`，同时新增 `--anim-fast/std/slow/ease` 全局动画变量。

```css
/* 之前：三个组件各自内联 */
@keyframes cs-bar  { ... }   /* CodeStats */
@keyframes token-bar { ... } /* Tokens */

/* 现在：统一为 */
@keyframes bar-reveal { from { clip-path: inset(0 100% 0 0) } ... }
@keyframes fade-up    { from { opacity:0; transform:translateY(4px) } ... }
@keyframes bar-grow   { from { transform: scaleY(0) } ... }
```
</ChangelogCard>

<ChangelogCard type="feat" title="环形缓冲区博客 + 交互可视化" hash="a7de334" date="2026-03-25">
移植并重写了 `RingBufferViz` 和 `GoGateBufferViz` 两个交互组件。`GoGateBufferViz` 支持手动 read / write，可以一步步看位运算取模怎么工作。写这个比写文章花时间多了。

```
    ┌───┬───┬───┬───┬───┬───┐
    │ 0 │ 1 │ 2 │ 3 │   │   │
    └───┴───┴───┴───┴───┴───┘
          ▲               ▲
        read            write

  capacity = 6  (must be power of 2)
  mask     = 5  (capacity - 1)
  index    = pos & mask  ← no division
```
</ChangelogCard>
